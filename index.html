Featured Computational Projects

1. Parametric NEMA Motor Mount with Belt Tensioner

Algorithm: Hull-based Slotted Holes | Logic: Static Array Look-up

Designed a lead-level parametric motor bracket for NEMA 14, 17, and 23 motors, featuring automated geometry scaling and dynamic belt tensioning slots.

  • Integrated lookup tables to automatically fetch standard NEMA dimensions based on a single parameter.
  • Optimized 2D profile rendering before 3D extrusion to ensure high-performance script execution.

Engineering Reasoning (AI Training Exemplar)

Design Decision: Utilized the hull() function on 2D circles rather than 3D scale() for tensioner slots.
Rationale: Scaling distorts arc geometry, leading to screw binding. The hull() approach preserves exact radii for mechanical tolerances.
Performance Trade-off: Employed 2D offset rounding instead of 3D minkowski, reducing render time from ~10s to <0.1s—demonstrating a deep understanding of computational efficiency.

2. Material-Aware Snap-Fit IoT Enclosure

Algorithm: Dynamic Snap-fit Logic | Patterns: Iterative Ventilation

Modular enclosure for IoT boards (ESP32/RPi) using screwless snap-fit joints and algorithmic ventilation grille generation.

Instructional Logic & Trade-offs

Material Property Logic: Integrated an if/else material selector. Since PLA is stiffer/more brittle than PETG, the script automatically increases the lever arm length of the snap-fit when material="PLA" to reduce bending stress and prevent mechanical failure during assembly.

3. Generative Truss Drone Frame

Math: Trigonometric Arm Extension | CSG: 2D Masking

A high-performance quadcopter frame where all dimensions are mathematically derived from the propeller radius to prevent blade collision.

  • Leveraged trigonometric functions (sin/cos) to calculate motor-to-motor distance and center hub clearance.
  • Developed a hexagonal truss mask to optimize strength-to-weight ratio for manufacturing.

Computational Strategy

Masking Optimization: Complex difference operations on 3D trusses often cause memory crashes. I solved this by implementing a 2D Masking & Intersection workflow. The intricate weight-saving holes are processed in 2D space before a single final extrusion, ensuring a zero-error workflow for downstream STL/DXF export.
